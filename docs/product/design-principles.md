# Design Principles

## Product Feel

Hudson Hustle should feel:
- original, not derivative
- warm and tactile
- editorial and deliberate
- map-first, never dashboard-first
- legible under active play

Hudson Hustle should not feel:
- like a SaaS admin tool
- like a generic board-game companion app
- like a poster site that happens to contain a game
- like a direct imitation of MTA or Ticket to Ride artifacts

## Core Priorities

Priority order:
1. gameplay clarity
2. map legibility
3. shell hierarchy and brand voice
4. expressive styling

If a design change improves mood but weakens play readability, reject it.

## Shell-First Rule

During `v2.1`, shell language should evolve first:
- setup screens
- room lobby
- multiplayer game shell
- local-play shell
- status and timer modules
- panels and cards

The map should absorb only careful, adjacent influence during this phase.

## Typography Rule

Typography should separate:
- authored display moments
- operational gameplay text

Use expressive type for:
- page titles
- section titles
- status headlines

Use practical type for:
- timers
- controls
- labels
- chips
- dense panel copy
- gameplay-adjacent information

## AI-Era Workflow Rule

Assume:
- coding is not the expensive part
- design judgment is the expensive part
- existing Hudson Hustle code and UI are assets, not obstacles

Therefore:
- use research and critique to narrow direction first
- validate one slice before broad rollout
- prefer reversible system evolution over broad speculative redesign

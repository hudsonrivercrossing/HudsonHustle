## Status

All Slice 4 issues are now implemented and should be tracked as `Done` on the roadmap board.

## Issue 1

### Title
Harden timed turn handoff from human timeout into server-owned bot turns

### Objective
- Ensure a timed human turn can expire and immediately continue through the normal server-controlled bot loop when the next active seat is a `bot`.

### Acceptance Criteria
- A human timeout can auto-complete and hand off into a legal `bot` turn without stalling.
- The next human seat still receives a valid new deadline.
- Existing human-only timer behavior remains unchanged.

### Validation
- deterministic server tests for timed timeout handoff

### Out Of Scope
- new bot-strength heuristics

## Issue 2

### Title
Resume persisted mixed rooms correctly when a server-owned bot seat is active

### Objective
- Ensure reloaded active rooms do not stay stuck when the restored active seat is server-owned.

### Acceptance Criteria
- Reloading a persisted mixed room with a `bot` active seat resumes normal server-owned play.
- Restored deadlines remain sensible for the next human-controlled turn.
- Human-only room restore behavior remains unchanged.

### Validation
- deterministic server tests for persisted-room resume behavior

### Out Of Scope
- new persistence schema changes

## Issue 3

### Title
Keep reconnect and disconnect semantics explicit and stable in mixed rooms

### Objective
- Prove that reconnect remains human-only while mixed rooms continue broadcasting correct public state.

### Acceptance Criteria
- Human seats can disconnect and reconnect in mixed rooms without corrupting room lifecycle.
- `bot` seats remain non-reconnectable and server-owned.
- Mixed-room public state stays understandable during reconnect/disconnect events.

### Validation
- server tests for reconnect/disconnect semantics
- browser verification for one mixed-room reconnect path

### Out Of Scope
- new reconnect token UX

## Issue 4

### Title
Add mixed-room lifecycle regression coverage for timed and reconnect behavior

### Objective
- Make the newly hardened mixed-room lifecycle hard to break accidentally.

### Acceptance Criteria
- Tests cover timeout handoff, persisted-room resume, and reconnect/disconnect paths.
- Existing Slice 1-3 regressions remain green.

### Validation
- server tests
- web multiplayer regression suite

### Out Of Scope
- UI redesigns unrelated to lifecycle behavior

## Issue 5

### Title
Document Slice 4 lifecycle hardening and follow-on boundaries

### Objective
- Keep docs aligned once mixed-room lifecycle hardening lands.

### Acceptance Criteria
- `v2.2` docs explain that mixed-room timer/reconnect behavior is now hardened.
- Docs do not imply bot seats are reconnectable.
- Slice 5 freeze work remains clearly separate.

### Validation
- doc review against shipped Slice 4 behavior

### Out Of Scope

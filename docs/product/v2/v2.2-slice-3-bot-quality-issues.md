## Issue 1

### Title
Strengthen starting-ticket keep heuristics for the baseline bot

### Objective
- Improve the bot’s initial ticket-confirmation logic so it keeps a more credible subset than “highest points only.”

### Acceptance Criteria
- Ticket keep logic stays deterministic.
- The bot prefers more coherent ticket sets over obviously awkward high-point combinations where appropriate.
- Existing ticket-confirmation flow remains legal and stable.

### Validation
- fixed-state bot-policy tests for multiple starting-ticket sets

### Out Of Scope
- explanation UI
- difficulty tiers

## Issue 2

### Title
Improve route-claim prioritization for the baseline bot

### Objective
- Make the bot prefer stronger ticket-aligned or strategically valuable claims over weaker generic affordable claims.

### Acceptance Criteria
- Claim choice stays deterministic.
- The bot prefers stronger ticket-progress claims in representative fixed states.
- The bot still avoids illegal or impossible claim attempts.

### Validation
- fixed-state tests for claim selection
- regression coverage for full turn progression

### Out Of Scope
- deep opponent modeling
- stochastic play variation

## Issue 3

### Title
Improve draw-card decisions for the baseline bot

### Objective
- Make draw behavior align better with near-term claim plans instead of acting as a generic fallback.

### Acceptance Criteria
- Draw choices stay deterministic.
- In representative fixed states, the bot draws toward plausible claim colors rather than arbitrary accumulation.
- Existing legal-action and turn-loop behavior stays intact.

### Validation
- fixed-state tests for draw decisions
- server regression coverage for bot turn completion

### Out Of Scope
- public seed controls
- difficulty settings

## Issue 4

### Title
Add quality-focused regression coverage for the stronger baseline bot

### Objective
- Prove that the stronger bot remains deterministic, legal, and not obviously self-destructive across the upgraded policy surface.

### Acceptance Criteria
- Tests cover stronger ticket-keep, claim, and draw scenarios.
- Existing Slice 1 and Slice 2 regressions remain green.
- The stronger policy does not weaken hidden-information boundaries.

### Validation
- server tests
- existing web multiplayer regression suite where relevant

### Out Of Scope
- reconnect/timer hardening
- player-facing copy changes by themselves

## Issue 5

### Title
Document the Slice 3 bot-quality baseline and follow-on boundaries

### Objective
- Keep docs aligned once the first competent baseline bot heuristics land.

### Acceptance Criteria
- `v2.2` docs describe the stronger baseline as one deterministic competent policy.
- docs do not imply difficulty tiers already exist.
- boundaries for Slice 4 and later remain explicit.

### Validation
- doc review against shipped Slice 3 behavior

### Out Of Scope

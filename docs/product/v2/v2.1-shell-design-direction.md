# V2.1 Shell Design Direction

## Summary

`v2.1` should be a shell-first design-system iteration, not a full map redesign.

Priority areas:
- typography
- setup and lobby layout
- game shell hierarchy
- banners and timer treatment
- cards, panels, and player-identity surfaces

## Typography
- Use a more distinctive display direction for major headings and section framing.
- Keep body and dense UI text calmer and highly legible.
- Maintain three text modes:
  - editorial
  - operational
  - game-data
- Do not use expressive display typography for dense gameplay facts.

## Layout Hierarchy
- Board remains the dominant visual plane.
- Side rail becomes a quieter but more intentional support surface.
- Status banner becomes a high-confidence top layer, not a leftover utility strip.
- Setup and lobby should feel composed and editorial, not form-first.

## Surface Hierarchy
- `Canvas surface`: product background/world
- `Stage surface`: board container
- `Rail surface`: private info and actions
- `Status surface`: turn state, connection, timer, lobby state
- `Artifact surface`: cards, tickets, chips, identity pieces

Differentiate surfaces primarily through:
- spacing
- tone contrast
- edge treatment
- shadow depth
- typography hierarchy

## Banners And Timers
- Build one coherent banner system for:
  - your turn
  - waiting
  - host/setup state
  - reconnecting / failed realtime
  - game over / transition states
- Timer should be visually separate and instrument-like.
- Escalate urgency cleanly without becoming noisy or game-show-like.

## Cards And Panels
- Panels should become quieter and more architectural.
- Cards should carry more identity than structural panels.
- Tickets should feel like strategic artifacts, not just rows of content.
- Private hand, market, and ticket surfaces should feel related, but clearly role-specific.

## Map Applicability Boundary

### Safe influence
- refined label typography
- calmer surrounding frame
- more intentional adjacency between map and shell
- better ownership badge integration

### Unsafe influence
- lowering route readability
- over-stylizing labels
- weakening color clarity
- turning the board into an editorial poster composition

## Recommended First Implementation Slice
- `status/banner system`

This is the best first slice because it:
- strengthens shell identity quickly
- affects both local and multiplayer
- is easier to validate than a broad shell rewrite
- does not force immediate map restyling
